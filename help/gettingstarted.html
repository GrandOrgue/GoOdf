Getting Started

GOODF is a Graphical User Interface(GUI) based program for creating the Organ Definition file (ODF) used by the GrandOrgue sample based pipe organ simulator. The ODF file contains information on which sample (.wav file) to play in response to the activation of defined stops, couplers, and keyboard keys. It allows the definition of windchestgroups on which to mount the pipes, and expression controls to control volume, and tremulants to introduce a periodic pitch and amplitude variation into the sample.

Finally, you can create one or more organ panels to display GUI elements with which the organist interacts on one or more panels. Not only Stops, Pistons, couplers. And keyboards, but also backgrounds, pictures of the actual organ, and labels can be included. It is possible to to define the look of each stop, coupler, button, and keyboard using images you provide yourself, or you can use the default images built into GrandOrgue.

All of these and more objects defined are defined within the ODF, together with relationships between the objects.

General advice

When using GOODF data is entered into data entry fields (checkboxes or radio buttons for Boolean; numeric entry fields for numbers, text entry boxes or pulldowns for text data or items with a limited number of defined choices) This data will not be processed and remembered unless you exit its data entry field. You cannot do this by pressing the ENTER key, you need to use the TAB key, or click on another data entry field or button within the same window In particular, data will not take effect if you close a popup window either with the Icons at the upper right provided by your operating system or by clicking on an exit button. Data will not take effect if you edit one object in its main editing window and then click on another object without exiting the data entry field to which you made changes
